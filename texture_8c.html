◆ texture_create()

resource_result_t texture_create ( const char *  name_,
texture_t **  texture_ 
)

texture_tのメモリを確保し、テクスチャ名称を初期化する(名称以外のフィールドは0, NULLで初期化)

Note
処理に失敗した場合、texture_の状態は不変
Parameters
[in]name_テクスチャ名称
[out]texture_メモリ確保、初期化対象テクスチャ構造体インスタンスへのポインタ
Return values
RESOURCE_INVALID_ARGUMENT以下のいずれか
  • name_ == NULL
  • texture_ == NULL
  • *texture_ != NULL
RESOURCE_LIMIT_EXCEEDEDメモリシステム使用可能範囲上限超過
RESOURCE_BAD_OPERATIONメモリシステム未初期化
RESOURCE_NO_MEMORYメモリ確保失敗
RESOURCE_OVERFLOW計算過程でオーバーフロー発生(名称文字列が長すぎる)
RESOURCE_SUCCESS処理に成功し、正常終了

◆ texture_destroy()

void texture_destroy ( texture_t **  texture_)

texture_が保持するリソースと、自身のメモリを解放し、NULLで初期化する

Note
2重destroyを許可し、texture_ == NULLまたは*texture_ == NULLの場合は何もしない
Warning
データ破損により、ピクセルデータリソースの破棄に失敗した場合は、処理を続行しtexture_自身のメモリを解放する(この場合ピクセルデータは再利用不可でメモリリークとなる)
Parameters
[in,out]texture_リソース解放対象構造体インスタンスへのダブルポインタ

◆ texture_pixel_load()

resource_result_t texture_pixel_load ( texture_t *  texture_,
const char *  filepath_,
const char *  extension_ 
)

テクスチャピクセルデータをロードする

Note
以下のテクスチャ名称は特殊(ビルトインテクスチャ)で、filepath_, extension_はNULLでok
  • test_texture_red
  • test_texture_green
  • test_texture_blue
filepath_の末尾には'/'を付加すること
extension_の先頭には'.'を付加すること
Parameters
[in,out]texture_ロード対象テクスチャ名称を格納するテクスチャ構造体インスタンスへのポインタ
[in]filepath_ファイルパス
[in]extension_ファイル拡張子
Return values
RESOURCE_INVALID_ARGUMENTtexture_ == NULL
RESOURCE_DATA_CORRUPTED以下のいずれか
  • texture_->name == NULL
  • 計算過程でのデータ破損
RESOURCE_BAD_OPERATION以下のいずれか
  • texture_->pixels != NULL
  • texture_->channel_count != 0
  • texture_->width != 0
  • texture_->height != 0
  • メモリシステム未初期化
RESOURCE_LIMIT_EXCEEDEDメモリシステムの使用可能範囲上限超過
RESOURCE_NO_MEMORYメモリ割り当て失敗
RESOURCE_OVERFLOW計算過程でオーバーフロー
RESOURCE_FILE_OPEN_ERRORファイルオープン失敗
RESOURCE_FILE_CLOSE_ERRORファイルクローズ失敗
RESOURCE_FILE_READ_ERRORBMPファイルのヘッダまたはピクセルデータの読み込みに失敗
RESOURCE_UNSUPPORTED_FILEサポート対象外のBMPファイル(DEBUG_BUILD or TEST_BUILDで詳細なログが出力される)
RESOURCE_SUCCESS処理に成功し、正常終了
